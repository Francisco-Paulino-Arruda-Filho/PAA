void buscar_linear(int *v, int n, int x) {
                    for (int i = 1; i <= n; i++) {
                        if (v[i] == x) {
                            return i;
                        }
                    }
                    return -1;
                }

\( \text{O que cada variável representa: n é o tamanho do vetor, v é o vetor e x o valor chave} \)

\( \text{Prova de corretude:} \)

\( \text{Invariante de laço:} \)

\( \text{\(i \leq n\)} \)

\( \text{Caso base:} \)

\( \text{Se } n = 1 \text{, então } v[1] = x \text{ e a função retorna 1, pois o elemento vai estar na posição inicial.} \)
\( \text{Assim essa sequência de comandos é repetida por todas as iterações, e se o valor de x estiver em \(v[n]=x\),} \)

\( \text{Caso indutivo:} \)

\( \text{Se \(n > 1\) e \(v[1]!=x\), comparação feita na linha 3 então o \(i\) é incrementado} \)
\( \text{então o vetor inteiro será percorrido, ou seja, enquanto i for menor ou igual a n, essa chegagem é feita na linha 2} \)

\( \text{Terminação:} \)

\( \text{O laço for é finito, pois i é incrementado a cada iteração e o laço termina quando i = n,} \text{Logo, a função sempre termina} \)
\( \text{Ou seja, como o número de interações só vai até n, então o algoritmo é da base de } \Theta(n) \)

\( \text{Argumentação para provar que o pior caso é \(\Theta(n)\) } \)

\( \text{O pior caso é quando o valor x não está no vetor, então o laço for é executado n vezes, ou seja, ele vai até o final em busca do elemento que não existe} \text{Logo, a função é \(\Theta(n)\).} \)
